# Lakes Client

Клиентская версия проекта на Next.js App Router, построенная на основе админ-проекта, но без административных CRUD-сценариев.

## Что входит в клиентскую сторону

- адаптивный хедер для desktop и mobile
- личный кабинет пользователя с аватаркой
- страница `Озёра` с картой MapLibre
- dropdown для выбора водоёма
- переход на страницу водоёма
- дашборд водоёма с паспортом, таблицей записей и графиком
- фильтры по году и по параметру измерения
- обновлённые `types` и `api` без admin-only логики

## Что удалено по сравнению с админкой

В клиентской версии **нет**:

- управления пользователями
- создания/редактирования/удаления водоёмов
- создания/редактирования/удаления измерений
- интерфейсов для администраторских действий

## API, которое использует клиент

### Аутентификация

- `POST /auth/login`
- `POST /auth/register` — клиент всегда отправляет роль `CLIENT`
- `POST /auth/refresh`
- один из маршрутов для профиля:
  - `GET /auth/me`, или
  - `GET /auth/profile`, или
  - `GET /users/me`

Если этих маршрутов нет, фронт пытается взять `id` пользователя из localStorage и делает `GET /users/:id`.

### Озёра и измерения

- `GET /water-bodies`
- `GET /water-bodies/:id`
- `GET /water-bodies/:id/measurements`

## Запуск

1. Создайте `.env.local`:

```env
NEXT_PUBLIC_API_URL=http://localhost:8000
```

2. Установите зависимости:

```bash
npm install
```

3. Запустите проект:

```bash
npm run dev
```

## Примечания

- карта использует `maplibre-gl`
- для корректной работы маркеров желательно, чтобы у водоёмов были `latitude` и `longitude`
- график строится только по числовым полям измерений
